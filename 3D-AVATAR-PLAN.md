# 3D Avatar Implementation Plan

## Goal
Replace the static image-based character system with real-time 3D animated avatars using GLB models. The avatars will have lip sync driven by TTS, facial expressions driven by emotion detection, idle animations (breathing, blinking), and head movement.

## Tech Stack for 3D Rendering

**SceneView** — Android Filament-based 3D rendering library for Jetpack Compose
- `io.github.sceneview:sceneview` — Compose-compatible 3D viewer
- Loads GLB/GLTF models natively
- Supports blend shape animation (for lip sync and expressions)
- Supports skeletal animation playback (for idle/walk anims)
- GPU-accelerated via Filament engine
- Provides `ModelNode` with `animate()` and blend shape control

### Step 5: Build 3D Avatar Renderer Component

Create `AvatarRenderer.kt` — a Composable that:
1. Loads a GLB model via SceneView
2. Sets up camera focused on the character's head/upper body
3. Sets up lighting (3-point lighting for flattering look matching the dark romantic theme)
4. Exposes controls for:
   - Blend shape animation (visemes for lip sync)
   - Blend shape animation (expressions: smile, frown, blink, brow raise)
   - Skeletal animation playback (idle breathing, head movement)
   - Head rotation (for emotion-based tilt)

### Step 6: Build Blend Shape Lip Sync Driver

Create `BlendShapeLipSync.kt` that maps our existing `LipSyncBridge.MouthShape` values to standard ARKit/Oculus viseme blend shape names:

| MouthShape | ARKit Blend Shape | Fallback Blend Shape |
|------------|-------------------|---------------------|
| CLOSED | viseme_sil / jawOpen=0 | - |
| OPEN_A | viseme_aa / jawOpen=0.7 | mouthOpen |
| NARROW_E | viseme_E / jawOpen=0.3 | mouthSmile |
| ROUND_O | viseme_O / jawOpen=0.5 | mouthFunnel |
| WIDE_W | viseme_U / jawOpen=0.4 | mouthPucker |
| TEETH_F | viseme_FF / jawOpen=0.2 | mouthClose |
| LIPS_M | viseme_PP / jawOpen=0.05 | mouthPress |
| TONGUE_L | viseme_TH / jawOpen=0.3 | tongueOut |

Not all models will have the same blend shapes. The driver should detect available blend shapes on the loaded model and use the best available mapping.

### Step 7: Build Expression Driver

Map `EmotionState` to blend shape combinations:

| Emotion | Blend Shapes |
|---------|-------------|
| HAPPY | mouthSmile=0.7, cheekSquint=0.3 |
| SAD | mouthFrown=0.5, browInnerUp=0.4 |
| FLIRTY | mouthSmile=0.4, eyeSquint=0.3, headTilt=5° |
| THINKING | browInnerUp=0.6, eyeLookUp=0.3 |
| SURPRISED | eyeWide=0.8, jawOpen=0.3, browUp=0.7 |
| LAUGHING | mouthSmile=1.0, jawOpen=0.4, cheekSquint=0.5 |
| SHY | mouthSmile=0.2, eyeSquint=0.5, headTilt=-8° |
| NEUTRAL | all=0 |

### Step 12: Handle Large Model Sizes
The models total ~296 MB which is too large for an APK. Options:
- **Option A**: Bundle only the smaller models (<15MB) in the APK, download larger ones on demand (like Gemma model)
- **Option B**: Use Android App Bundle (AAB) with asset packs
- **Option C**: Compress/optimize the GLB files (many can be significantly reduced with Draco compression via `gltf-transform`)
- **Recommended**: Option C first (compress all models), then Option A for any still over 20MB
